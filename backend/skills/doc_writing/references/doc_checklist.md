# 문서 작성 체크리스트

> 문서 완성 전 반드시 확인하세요.

---

## ✅ PRD 체크리스트

### 기본 정보
- [ ] 문서 제목이 기능/제품명을 명확히 나타내는가?
- [ ] 작성일, 버전, 작성자, 상태가 기재되어 있는가?
- [ ] 변경 이력 테이블이 있는가?

### 내용 완결성
- [ ] 배경과 문제 정의가 구체적인가? (수치 또는 사례 포함)
- [ ] 정량적 목표가 명시되어 있는가?
- [ ] 개발 범위(포함/미포함)가 명확히 구분되어 있는가?
- [ ] 주요 사용자 유형이 정의되어 있는가?
- [ ] Must Have / Should Have / Nice to Have 기능이 구분되어 있는가?
- [ ] 각 기능에 수용 기준(Acceptance Criteria)이 있는가?
- [ ] 비기능 요구사항(성능, 보안 등)이 포함되어 있는가?

### 연계 정보
- [ ] 관련 디자인 링크(Figma 등)가 첨부되어 있는가?
- [ ] API 명세가 필요한 경우 포함되어 있는가?
- [ ] 일정과 마일스톤이 있는가?
- [ ] 위험 요소와 대응 방안이 있는가?
- [ ] 열린 질문(Open Questions)이 정리되어 있는가?

---

## ✅ 정책 문서 체크리스트

- [ ] 정책 적용 대상이 명확한가?
- [ ] 정책 시행일이 명시되어 있는가?
- [ ] 예외 사항이 정의되어 있는가?
- [ ] 위반 시 처리 방안이 있는가?
- [ ] 정책 개정 이력이 있는가?
- [ ] 담당 부서/담당자가 명시되어 있는가?

---

## ✅ 회의록 체크리스트

- [ ] 회의 일시, 장소(온/오프라인), 참석자가 기록되어 있는가?
- [ ] 안건(Agenda)이 사전에 정의되어 있었는가?
- [ ] 각 안건별 논의 내용이 요약되어 있는가?
- [ ] **결정 사항**이 명확히 구분되어 있는가?
- [ ] **액션 아이템**에 담당자와 완료 기한이 있는가?
- [ ] 다음 회의 일정이 포함되어 있는가?

## ✅ 기술 기획서 체크리스트

- [ ] 시스템 아키텍처 다이어그램이 있는가?
- [ ] 기술 스택 선정 이유가 설명되어 있는가?
- [ ] 데이터 모델/스키마가 정의되어 있는가?
- [ ] API 엔드포인트 목록이 있는가?
- [ ] 외부 시스템 연동 방법이 설명되어 있는가?
- [ ] 보안 고려 사항이 포함되어 있는가?
- [ ] 성능/부하 요구사항이 있는가?
- [ ] 배포 방법 및 인프라 구성이 설명되어 있는가?

---

## ✅ 공통 최종 점검

### 가독성
- [ ] 문장이 너무 길지 않은가? (한 문장 = 한 아이디어)
- [ ] 전문 용어에 설명이 붙어 있는가?
- [ ] 헤딩 구조가 논리적인가?
- [ ] 표와 목록이 적절하게 활용되었는가?

### 정확성
- [ ] 수치/데이터의 출처가 명시되어 있는가?
- [ ] 날짜 형식이 통일되어 있는가? (YYYY-MM-DD 권장)
- [ ] 오탈자 검토를 했는가?

### 완결성
- [ ] 독자가 추가 질문 없이 내용을 이해할 수 있는가?
- [ ] 참조 문서/링크가 모두 유효한가?
- [ ] 다음 단계(Next Steps)가 명확한가?

## 📊 문서 품질 점수 기준

| 점수 | 기준 |
|---|---|
| 90점 이상 | 배포 가능. 체크리스트 항목 10% 미만 미충족 |
| 70~89점 | 보완 필요. 핵심 항목은 충족, 부가 항목 보완 |
| 70점 미만 | 재작성 권고. 핵심 섹션 누락 |

> **핵심 항목**: 목적, 범위, 요구사항, 담당자, 일정
